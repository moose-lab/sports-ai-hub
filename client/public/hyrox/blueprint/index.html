An open build lane for hybrid fitness racing. HYROX ships a fixed format, timed to the second — a sport that emits structured data by design, with almost no open-source tooling on the shelf. 01

Why HYROX is a build target

The pitch

Most sports need computer vision just to extract data. HYROX emits it. Three properties make this sport the best open-source whitespace in fitness right now.

Property 1

A sport with a schema

Fixed course, fixed station order, fixed movement standards, divisions with published loads. Every finisher produces the same structured record: 8 run splits, 8 station splits, roxzone time. Tools generalize instantly — no per-team data wrangling.

Property 2

A crowd that measures itself

Billed as the world's fastest-growing mass-participation race format: dozens of city events per season, hundreds of thousands of entries, races that routinely sell out. And the athletes are the tool users — they hunt for pacing plans, split benchmarks, and sled-weight tables every single week.

Property 3

An empty shelf

Search GitHub for running or cycling and you get tens of thousands of repos. Search hyrox and you get a shelf you can still be first on. Nearly every tool niche below is unclaimed — the first decent repo owns its keyword.

Search HYROXCN-certified gyms by browser location, coordinates, or region keyword — or import a HYROXCN JSON export. Ranked gym list, distance summary, booking and fitness-test flags, lightweight map. Already listed in the catalog and the README's Open-Source Projects section.

Input

Output

05

The dual flywheel

Collect ⇄ be collected

An awesome list and its listed tools are not separate products. The list is a distribution channel; the small repos are its long-tail SEO. Wire them together and every shipped tool pulls the next one up.

1 · List
Zone lists a tool — catalog entry, README line, release note.
2 · Badge
Tool's README carries the "listed in" badge, linking back.
3 · Rank
Exact-match repo name ranks for its niche keyword on GitHub & Google.
4 · Land
Athletes and devs land on the tool, follow the badge to the zone.
5 · Star
They star and watch. Stars surface the repo in followers' feeds.
6 · Build
Good-first-issues convert visitors into builders. New tools fill gaps.
↩ every new tool re-enters at step 1 — the loop compounds
  • One tool = one keyword = one audience door. Never bundle three ideas into one repo; you'd collapse three doors into one.
  • Stars follow usefulness density, not effort. A 200-line split parser that answers "was my time good?" outperforms a half-finished platform. Curate ruthlessly.
  • The zone gives distribution, tools give proof. A list with live, runnable entries converts visitors; a list of links does not.
06

GitHub SEO & keyword capture

Name · About · topics · README

GitHub search weighs repo name, then About description, then topics and README. Google independently indexes READMEs and topic pages. So the fastest exposure play is mechanical: put the phrase people type in the fields machines read first.

Doctrine 1

Exact-match names

hyrox-pacing-calculator beats paceforge in search every day of the week. Save the clever brand for v2 — the repo name is your headline keyword slot.

Doctrine 2

The About formula

Verb + object + audience + differentiator, keywords front-loaded, ≤ 160 chars. "Find HYROX-certified training gyms near you…" — not "A modern web app built with…".

Doctrine 3

Topic pages are shelves

Every topic creates a browsable, Google-indexed page. github.com/topics/hyrox has almost nothing on it — being early on an empty shelf is free real estate. Up to 20 topics per repo; exact lists in the Locker.

Last placement rule: README H2s should be phrases people type — "HYROX pacing", "HYROX results analysis" — not "Our philosophy". Google shows those headings as jump links in results.

07

Star & save mechanics

6 levers, all GitHub-native

Ranked by leverage for a young awesome list. Every lever below is a norm of the ecosystem — nothing here games a metric.

1

Awesome-ecosystem cross-listing highest leverage

The awesome ecosystem is pre-aggregated demand: get listed where list-seekers already are. Requirements for the main sindresorhus/awesome index: ~30 days of public history, clean npx awesome-lint run, the awesome badge, and contribution docs — this repo already carries the badge, CONTRIBUTING, and CC0.

Also PR into adjacent lists (awesome-sports-analytics is already in this catalog — propose a reciprocal listing) — but only where genuinely on-topic. Off-topic submissions burn maintainer trust.

2

The badge exchange the flywheel's return path

Every listed tool is offered a "listed in awesome-sports-ai" badge (markdown in the Locker). It's the mechanism that turns each tool's traffic into zone discovery. Offer it on the listing PR; always optional for the tool owner.

3

Release cadence as broadcast monthly

GitHub's home feed shows releases and stars from repos people follow — every stargazer becomes a broadcast node, but only if you cut releases. Tag a monthly zone release ("HYROX Zone v0.2: split parser ships, +4 tools") with human-written notes. Template in the Locker.

4

Good-first-issue funnel contributor engine

Seed 10 concrete starter issues (list in the Locker), label them good first issue + help wanted. GitHub's own /contribute page lights up automatically, and aggregators like up-for-grabs.net accept project registrations by PR. Each converted contributor is a star, a watcher, and a promoter.

5

Concentrated launch windows rhythm

Feeds and trending surfaces respond to velocity, not totals. Batch announcements — zone launch, tool ship, awesome-list merge — into the same 24–48 h window instead of dripping them. One loud, honest day beats five quiet ones.

6

Proof surfaces conversion

A star-history chart in the README, a contributors grid, a "used by" line on the gym finder. Social proof converts the lurker who found you via search into a stargazer. All embeds are free (star-history.com, contrib.rocks).

Anti-pattern, for the record: star-exchange rings and bought stars. GitHub prunes them, trending filters for account quality, and a pruned spike reads as fraud in your own analytics. Velocity must come from batching honest news.

## HYROX Zone

*The first specialty sport zone: build, list, and use tools for hybrid fitness racing. Scene data lives in [`data/catalog.json`](data/catalog.json); the full scenario guide is [docs/sports/hyrox.md](docs/sports/hyrox.md).*
